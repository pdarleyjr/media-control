## Mocks Remaining

The following mocks remain, but ONLY when `globalThis.__MC_ENTERPRISE_MOCK_ONLY === true`:

1. `ROOMS_MOCK` — rooms catalog fixture (test harness only)
2. `MOCK_CONTENT_FIXTURE` — content library fixture (test harness only)
3. `MOCK_SCREENSHARE_DIAGNOSTICS` — degraded screen-share diagnostics fixture (test harness only)
4. `privacy.requestOrganizationPublication` / `privacy.setVisibility` — fake success (test harness only)

## Proof Mocks Cannot Run in Production

1. `MOCK_ONLY` is evaluated at module import time from `globalThis.__MC_ENTERPRISE_MOCK_ONLY`.
2. No production code path sets this flag. It is set ONLY by:
   - The Playwright fixture (`console.html`) via a classic `<script>` tag before module imports
   - Node tests that explicitly set `globalThis.__MC_ENTERPRISE_MOCK_ONLY = true` before importing
3. Five production-safety tests (`enterprise-api.test.js` tests 6-10) prove that in production mode (flag unset), every previously-mocked method either throws an explicit error or propagates the real failure — never returns mock data or fake success.
4. The flag is a runtime global, not an env var or build flag — it cannot be set by deployment configuration.
