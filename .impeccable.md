# MBFD Media Control — Impeccable Design System

> Created 2026-06-02 | Design Context for AI design/UI skills (frontend-design, polish, critique, anti-slop-ship, etc.)

---

## Design Context

### Users
**Classroom-first.** The primary operator is a training instructor at a podium driving a video wall or classroom displays during presentations, lectures, whiteboard sessions, YouTube/video, and screen-share. They are standing, sometimes gloved, often non-technical, and acting live in front of a room. A secondary surface is the unattended **display/player** (video walls, conference-room screens). Benchmark against Command 360 and classroom-AV peers (Vivi, Solstice, ClickShare, SMART), not EOC dashboards.

### Brand Personality
**Authoritative. Trusted. Calm under live pressure.** Same Miami Beach Fire Department identity as the Hub, expressed for a classroom-AV control surface: legible at a glance, forgiving to operate live, never busy or playful. Confidence, not spectacle.

### Aesthetic Direction
- **Light everywhere** (2026-05-30 redesign). White cards on a warm off-white page, warm near-black text.
- **Go vs Live semantic is sacred**: **blue = go / interactive** (the affordance you press), **saturated red = on-air / live / destructive only**. Never use saturated red for decoration or generic accents.
- **One disciplined CTA** (orange `--cta`) for the single dominant action on a surface (Go Live / Present). Never status-coded.
- Anti-references: dark "command bunker" theming, neon, busy dashboards, tiny controls.
- Emotional goal: an instructor feels in control and unhurried; a wrong tap is obvious and reversible.

## Stack & Implementation Constraints (emit code for THIS stack)

- **Vanilla JavaScript, ES modules. NO framework, NO build-step React, NO JSX, NO Tailwind.** Do not introduce React/Vue/Svelte or a bundler-only API.
- **Views**: `frontend/js/views/*.js` — each exports `render()`/`mount()` returning DOM strings/nodes. **Components**: `frontend/js/components/*.js` (toast, confirm, modals). Router: `frontend/js/app.js`.
- **Output framework-free DOM**: template strings + `document.createElement`, wired with `addEventListener`. Escape interpolated content with the existing `esc()` helper.
- **Styling = plain CSS with CSS custom properties** in `frontend/css/{variables.css, main.css, ...}`. Style by editing/using CSS-variable tokens, NOT inline magic numbers and NOT utility classes. ~367 token-driven rules flip cohesively off these vars.
- **i18n is mandatory**: route every user-visible string through `t()` / `tn()` (8 languages in `frontend/js/i18n/`). NEVER hardcode display strings.
- **Realtime** via socket (`on()`/`off()`); toasts via `showToast()`.
- New "Studio" surfaces (Home dashboard, slide editor, broadcast center) render on `.mc-studio-surface` and use the additive `--mc-*` token namespace; existing classroom views (Present, Displays, Scenes, Whiteboard) use the base tokens and must keep rendering unchanged.

## Spacing, Type, Touch (base tokens)
- Spacing: `--space-1..8` (4/8/12/16/24/32/48/64px), 4px rhythm.
- Type: `--font-1..8` (11..40px); `--font-8` for big glanceable room/status headers.
- **Touch targets**: `--tap-min: 48px` everywhere, `--tap-primary: 64px` for primary live actions. The instructor surface uses these at ALL widths (standing, possibly-gloved user). Never go below 48px.

---

## Critical Rules (Violations = Bugs)

| Rule | Why |
|---|---|
| NEVER hardcode visible strings — use `t()`/`tn()` | 8-language i18n; hardcoding breaks localization |
| Blue = go, saturated red = live/destructive ONLY | Classroom go-vs-live safety semantic; a red "press me" is a live-air hazard |
| Live/on-air state NEVER color-alone — pair label or icon | WCAG 1.4.1; instructor must read state at a glance and color-blind-safe |
| Edit CSS-variable tokens, not inline magic numbers | 367 rules flip cohesively off the vars; inline values fragment the theme |
| No framework / no Tailwind / no JSX | App is vanilla ES modules; adding a framework breaks the build and the pattern |
| Touch targets >= 48px (64px for primary live actions) | Standing, gloved, non-technical live operator |
| Escape interpolated content (`esc()`); animate transform/opacity; gate `prefers-reduced-motion` | XSS safety + performant, accessible motion |
| One dominant `--cta` per surface; never status-code the CTA | Operator must instantly find THE action |

## States (every interactive surface)
- **Loading**: skeleton/placeholder, never a bare spinner-only screen.
- **Empty**: composed message routed through `t()` (e.g. "no displays connected"), never blank.
- **Error**: inline, human, via `showToast()` or a panel; never a raw error string or `window.alert`.
- **Live/Standby/Frozen**: always a token color PLUS a label/icon.
